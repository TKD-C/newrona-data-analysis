##과제
1. 경기 기록시 elo(점수) 증감 방법 변경 :
- 현재 : 1대1 적 라이너와 비교한다음 팀 평균elo로 계산
- 변경 : 팀 평균 elo끼리 비교하여 예상 승률에 따라 팀원 전체 같은 양으로 증감, 그리고 divisor가 현재 4000인데 => 600으로 바꾸기

2. 팀 짜줄때 주라인 위주로 짜주기
- 어쩔수 없으면 부라인으로 짜주는데, 그러면 elo를 낮춰서 계산 (감소폭(l) = 50) 
- 부 라인으로 가게 된 내전러에게 "반창고"(int) 1개추가. 
- 팀을 짤때 반창고가 많은 사람 부터 주 라인이됨
- 주 라인은 최대 2개라서, 주 라인이 2개인 사람은 비는 라인쪽으로 : 예) 주라인 탑(1개)4명 탑,원딜(2개)1명 이면 주라인 탑원딜이 원딜가게됨 
-  반창고 이모지 =  :adhesive_bandage: 
##과제 2: 주라인 위주 팀짜기 + 반창고

  방향 좋은데, 지금 명세로는 바로 구현 못 합니다. 결정해야 할 것들:

  (1) 반창고 수명 — 가장 중요
  명세에 "부라인 가면 +1", "반창고 많은 사람부터 주라인"은 있는데 언제 줄어드는지가 없습니다. 영구 누적이면 한 번 불운한
  사람이 영원히 우선권을 독점합니다.
  - 추천: 부라인 배정 시 +1, 주라인 배정 시 −1(최소 0). 그래야 "빚 갚기" 구조로 순환됨.
  - 그리고 언제 적립? 팀 편성하는 순간인지(/팀짜주기 실행 시) vs 경기 기록될 때인지. → 편성 순간 추천(즉각적, 명확).
=> 경기 기록될 때
  (2) 저장 위치
  반창고는 영구 상태라 Player에 Bandage(int) 필드 추가 + Firestore players/{id}.bandage 저장 필요 (additive).
  마이그레이션은 없는 필드 기본 0이면 됨.

  (3) 우선순위 충돌
  세 목표가 서로 부딪힙니다. 순서를 박아야 합니다:
  1. 주라인 커버 최대화(부라인 인원 최소화)
  2. 그래도 부라인 가야 하면 → 반창고 많은 사람이 주라인
  3. 그 다음 Elo 균형(부라인은 −50 적용한 가상 점수로)
  - 이 순서가 맞나요? (예: 균형을 위해 반창고 적은 사람을 억지로 주라인 줘야 하는 상황이면 누가 이기나)
=> 이 순서가 맞다. (라인을 5대5 정하고, elo맞추기)
  (4) 부라인으로도 못 채우는 라인
  아무도 정글 주/부가 없으면 그 라인은 누가 가나요? 강제 배정 + 추가 페널티? 규칙 필요.
=> 강제 배정 후 (-150 점수 = 3(l))
  (5) −50(l) 적용 범위 확인
  "부라인이면 Elo 낮춰 계산"은 팀 균형 맞추기용 가상 점수만 낮추는 거죠? 실제 저장 Score는 안 건드리는 게 맞다고 봅니다.
  (확인)
=> 맞음

  (6) 어느 명령에?
  지금 /팀짜주기뱀·/팀짜주기랜덤은 "라인 무시" 컨셉입니다. 라인 기반은 성격이 달라서:
  - 추천: 새 명령 /팀짜주기라인 추가, 기존 둘은 유지.
=> 이거 좋다
  - 단, 라인 기반이 제대로 돌려면 참가자들의 MainLanes/SubLanes가 채워져 있어야 합니다. 비어 있는 내전러가 많으면
  fallback 투성이가 됨 — 데이터 채우기가 선행돼야 함.

  (7) 알고리즘 난이도
  10명을 5라인×2팀에 배정 = 제약 충족 + 최적화입니다. 10명이면 전수 탐색 가능한 규모라 구현은 됩니다만, (3)의 우선순위를
  점수화(예: 부라인 1명당 페널티 P, 반창고 위반 페널티 Q, Elo차)해서 최소비용 배정을 고르는 식으로 짜야 깔끔합니다.
